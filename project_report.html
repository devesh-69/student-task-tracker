CHAPTER 1: INTRODUCTION

1.1 Introduction of the Project (SRS)

In today’s fast-paced academic environment, students often struggle to keep track of their numerous assignments, projects, and study deadlines. The Student Task Tracker is a web-based application developed to solve this problem. It is a "Single Page Application" (SPA) built using React.js that allows students to organize their academic life efficiently.

The project leverages Current Trends in IT, specifically Generative AI (via the Gemini API) to help students plan their work, and Reactive Web Architecture to provide a seamless, app-like experience without page reloads. The system is designed to be "Offline-First," storing data securely on the user's device using LocalStorage.

1.2 Problem Definition

The core problems faced by students that this project addresses are:

  • Disorganization: Students rely on scattered notes, physical diaries, or memory to track deadlines, leading to forgotten assignments.
  • Lack of Planning: Even if a task is noted, students often don't know "how" to start. They lack a breakdown of complex tasks.
  • Poor Visualization: Traditional to-do lists are static text. They do not show "how much" of a task is done (e.g., 50% vs 90%).

1.3 Objective of Project

The primary objectives of the Student Task Tracker are:

  • To develop a user-friendly interface for Creating, Reading, Updating, and Deleting (CRUD) academic tasks.
  • To integrate Artificial Intelligence to suggest sub-tasks and breakdowns for study topics.
  • To implement a visual Progress Tracking system (Interactive Sliders and Progress Bars).
  • To use Gamification (Audio effects and Animations) to motivate students upon task completion.
  • To ensure data persistence without requiring a complex backend server setup.

1.4 Scope of Project

The scope of this project is focused on a Personal Academic Manager for a single user.

  • In Scope: Task management, AI study planning, Deadline tracking, Local data storage, Responsive mobile design.
  • Out of Scope: Multi-user collaboration, Cloud database synchronization, Payment gateways.

CHAPTER 2: SYSTEM STUDY

2.1 Existing System

Currently, students use methods such as:

  • Physical Planners/Diaries: These are prone to damage and loss. They offer no reminders or automation.
  • Generic Note Apps (e.g., Notepad, Keep): These apps store text but do not understand "Dates" or "Progress". They cannot calculate "Days Remaining" or show progress bars.

2.2 Disadvantages of Existing System

  • No Intelligence: A physical diary cannot tell you how to solve a math assignment; it only records that you have one.
  • Static Data: You cannot interactively slide a progress bar on paper.
  • Accessibility: Paper notes are not accessible in dark environments or if left at home.

2.3 Proposed System

The Student Task Tracker digitizes the entire process. It transforms a static "To-Do List" into a dynamic "Task Dashboard".

Key Advantages:

  • AI-Powered: The system acts as a study companion, suggesting how to approach tasks.
  • Visual & Interactive: Users can visually see their workload and interact with tasks using touch gestures (Long Press) and sliders.
  • Responsive: Works perfectly on Mobile phones and Laptops.

2.4 Use Cases

The following actors and use cases define the system interaction:

  • Actor: Student (Devesh)
  • Use Case 1 - Add Task: The student inputs a task title (e.g., "Mini Project Report"). The system saves it.
  • Use Case 2 - Get AI Help: The student clicks "AI Assist". The system uses Gemini API to break the task into steps (e.g., "1. Draft Introduction, 2. Design Diagrams...").
  • Use Case 3 - Track Progress: The student slides the progress bar to 50%. The system logs "Progress updated to 50%".
  • Use Case 4 - Celebration: When progress hits 100%, the system plays a sound and shows a confetti animation.

3.2 Technologies Used (Tech Stack)

This project utilizes a modern "MERN-style" frontend stack:

  • React.js (Frontend Framework): Used for building the component-based UI. React allows the app to be fast and responsive, updating only parts of the screen that change.
  • TypeScript: Used to add static typing to JavaScript, reducing bugs and improving code quality.
  • Tailwind CSS: A utility-first CSS framework used for styling. It ensures the app looks great on mobile devices (Responsive Design).
  • Google Gemini API: The Generative AI model used to provide intelligent study suggestions.
  • LocalStorage API: A browser-native database used to persist user data permanently on the device without needing an internet connection for storage.

CHAPTER 5: USER MANUAL

5.1 Explanation of Key Functions

1. Onboarding

When you open the app for the first time, a popup will ask for your Name. Enter your name to personalize the dashboard.

2. Adding a Task

Click the circular Floating Button (+) at the bottom right. A form will appear. Enter the Task Title and Deadline. If you are unsure how to start, click the "AI Assist" button to let the AI write a plan for you.

3. Updating Progress

Click on any task card. A detail window will open. Drag the Slider to update your percentage (e.g., from 0% to 50%). You can also type a log message (e.g., "Finished Chapter 1") to keep a history of your work.

4. Editing and Deleting

To Edit or Delete, Long Press (Hold) on a task card for 1 second. A menu will pop up giving you options to modify or remove the task.

CHAPTER 6: OBSTACLES FACED

During the development of this Mini Project, I faced several technical challenges which I had to overcome:

1. Handling Mobile Gestures (Long Press)

Challenge: Browsers do not have a native "Long Press" event. The standard onClick event conflicts with holding down a button.
Solution: I implemented a custom logic using onTouchStart, onTouchEnd, and JavaScript Timers to detect if a user is holding the screen for more than 500ms.

2. Integrating AI securely

Challenge: Connecting to the Gemini API required handling asynchronous network calls and potential errors if the internet was slow.
Solution: I created a dedicated geminiService.ts file to handle all AI errors gracefully and provide fallback messages if the AI is unreachable.

3. State Synchronization

Challenge: When I updated a task inside the "Detail Modal", the main "Dashboard List" didn't update immediately.
Solution: I used React's State Management to ensure a "Single Source of Truth", so changes in one component instantly reflect everywhere.
